### 3. 改进 Prompt 工程 ✅

**目标**: 解决测试中发现的命令质量问题，提升生成准确性

**问题分析**:
1. 复杂命令混用 Windows 和 Linux 命令
2. 输出包含解释文字，不符合"只输出命令"的要求
3. 缺少明确的操作系统环境约束

**改进措施**:

#### 命令生成 Prompt 改进
- 添加【关键要求】章节，强化输出格式约束
- 提供错误示例 vs 正确示例对比
- 明确操作系统和 Shell 环境
- 改进 Windows 命令示例（添加 forfiles、tasklist 等）
- 强调不要混用不同操作系统的命令
- 明确路径分隔符规范

#### 代码生成 Prompt 改进
- 添加具体代码示例（Python 和 Shell）
- 明确代码规范要求（PEP 8、错误处理等）
- 强调最佳实践和错误处理

#### 输出清理功能
- 新增 `clean_command_output()` 方法
- 自动过滤解释文字（"注意："、"说明："、"提示："等）
- 跳过注释行（#、//）
- 只返回第一行有效命令

**改进效果**:

| 测试场景 | 改进前 | 改进后 | 状态 |
|---------|--------|--------|------|
| 查找大于 1MB 的文件 | 混用 Windows/Linux 命令 | `forfiles /S /C "cmd /c if @fsize gtr 1048576 echo @path"` | ✅ |
| 列出 Python 文件 | - | `dir /s /b *.py*` | ✅ |
| 显示磁盘使用 | 包含大量解释文字 | `dir /-c /w /s /b` | ✅ |

**提交**: `e6824d9 feat: 改进 Prompt 工程，提升命令生成质量`
